# Algorithm Dinic — Maximum Flow Benchmark

Проект посвящён задаче нахождения максимального потока в транспортной сети и сравнению трёх классических алгоритмов:

- Ford-Fulkerson
- Edmonds-Karp
- Dinic

Основной алгоритм проекта — **алгоритм Диница**.

---

## Описание проекта

Задача максимального потока заключается в нахождении максимального количества ресурса, которое можно передать из источника `s` в сток `t` по ориентированной сети с ограниченными пропускными способностями рёбер.

В проекте реализованы три алгоритма максимального потока и проведено сравнение их времени работы на случайно сгенерированных графах.

---

## Алгоритмы

### Ford-Fulkerson

Базовый алгоритм поиска максимального потока.

Идея:

- искать увеличивающий путь;
- проталкивать по нему поток;
- повторять, пока путь существует.

Недостаток: время работы зависит от выбора путей и величины максимального потока.

---

### Edmonds-Karp

Улучшение алгоритма Форда — Фалкерсона.

Идея:

- искать кратчайший увеличивающий путь с помощью BFS;
- увеличивать поток по одному пути за раз.

Сложность:

```text
O(VE²)
```

---

### Dinic

Алгоритм Диница использует:

- остаточную сеть;
- слоистую сеть;
- блокирующий поток.

Идея:

- BFS строит уровни вершин;
- DFS проталкивает поток по слоистой сети;
- за одну фазу используется несколько путей.

Сложность:

```text
O(V²E)
```

## Наследование

В проекте используется наследование.

Базовый класс:

```cpp
class MaxFlowSolver {
public:
    virtual int solve() = 0;
};
```

От него наследуются классы алгоритмов:

```cpp
class FordFulkersonSolver : public MaxFlowSolver
class EdmondsKarpSolver : public MaxFlowSolver
class DinicSolver : public MaxFlowSolver
```

Каждый алгоритм реализует свой метод:

```cpp
int solve() override
```

## Генерация графов

Для тестирования создаются случайные ориентированные графы.

Особенности генерации:

- гарантируется путь от источника к стоку;
- добавляются случайные рёбра в разных направлениях;
- пропускные способности задаются случайно;
- количество дополнительных рёбер равно `8n`.

Общее количество логических рёбер:

```text
E = 9n - 1
```

где `n` — количество вершин.

## Результаты

В эксперименте алгоритмы сравниваются на графах разного размера.

Ожидаемый результат:

- Ford-Fulkerson работает медленнее на больших графах;
- Edmonds-Karp работает стабильнее;
- Dinic показывает лучшее время на сложных графах.

Главная причина эффективности Dinic:

> алгоритм строит слоистую сеть и ищет блокирующий поток, поэтому за одну фазу может использовать сразу несколько путей.

В проекте реализованы и сравнены три алгоритма нахождения максимального потока.

Алгоритм Диница показывает лучшие результаты на больших и сложных графах благодаря использованию слоистой сети и блокирующего потока.
