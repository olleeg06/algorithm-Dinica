# Dinic Maximum Flow Benchmark

Проект посвящён задаче нахождения максимального потока в транспортной сети и сравнению трёх классических алгоритмов:

- Ford-Fulkerson;
- Edmonds-Karp;
- Dinic's algorithm.

Основной алгоритм проекта — **алгоритм Диница**. Алгоритмы Форда-Фалкерсона и Эдмондса-Карпа используются как базовые методы для сравнения времени работы.

## Цель проекта

Цель проекта — реализовать алгоритм Диница в объектно-ориентированном стиле на C++, проверить корректность реализации и сравнить его производительность с другими алгоритмами максимального потока.

## Постановка задачи

Дана транспортная сеть: ориентированный граф с источником `s`, стоком `t` и пропускными способностями рёбер.

Необходимо найти максимальный поток из `s` в `t`, соблюдая условия:

1. поток по ребру не превышает его пропускную способность;
2. поток неотрицателен;
3. для всех промежуточных вершин входящий поток равен выходящему.

## Алгоритмы

### Ford-Fulkerson

Базовый метод, который ищет увеличивающие пути и постепенно увеличивает поток.

Преимущество — простота реализации. Недостаток — время работы зависит от выбора увеличивающих путей и величины максимального потока.

### Edmonds-Karp

Улучшение метода Форда-Фалкерсона. Алгоритм использует BFS для поиска кратчайшего увеличивающего пути.

Сложность:

```text
O(VE^2)
```

### Dinic's Algorithm

Алгоритм Диница строит слоистую сеть с помощью BFS, а затем ищет блокирующий поток с помощью DFS.

Сложность в общем случае:

```text
O(V^2E)
```

где:

- `V` — количество вершин;
- `E` — количество рёбер.

## Основная идея алгоритма Диница

Алгоритм Диница работает фазами:

1. BFS строит уровни вершин в остаточной сети.
2. DFS проталкивает поток только по рёбрам, ведущим на следующий уровень.
3. В текущей слоистой сети ищется блокирующий поток.
4. После изменения остаточной сети процесс повторяется.
5. Если сток больше недостижим из источника, поток является максимальным.

Краткая схема:

```text
max_flow = 0

while BFS(source, sink):
    ptr[v] = 0 for every vertex

    while pushed = DFS(source, INF):
        max_flow += pushed

return max_flow
```

## Объектно-ориентированная структура

В проекте используется ООП. Основные сущности вынесены в отдельные классы.

| Класс | Назначение |
|---|---|
| `Edge` | Хранит ребро: конечную вершину, индекс обратного ребра и остаточную пропускную способность |
| `FlowNetwork` | Хранит транспортную сеть и список смежности |
| `MaxFlowSolver` | Базовый абстрактный класс для алгоритмов максимального потока |
| `FordFulkersonSolver` | Реализация алгоритма Форда-Фалкерсона |
| `EdmondsKarpSolver` | Реализация алгоритма Эдмондса-Карпа |
| `DinicSolver` | Реализация алгоритма Диница |
| `GraphGenerator` | Генерация случайных тестовых графов |
| `Benchmark` | Запуск тестов, измерение времени и сохранение результатов |

Наследование используется в классах алгоритмов:

```cpp
class FordFulkersonSolver : public MaxFlowSolver { ... };
class EdmondsKarpSolver : public MaxFlowSolver { ... };
class DinicSolver : public MaxFlowSolver { ... };
```

Все алгоритмы имеют общий метод:

```cpp
int solve() override;
```

Это позволяет запускать разные алгоритмы через общий интерфейс.

## Генерация графов

Для каждого значения `n` создаётся случайный ориентированный граф.

В генераторе:

- создаётся гарантированный путь `0 -> 1 -> 2 -> ... -> n - 1`;
- добавляются случайные рёбра в разных направлениях;
- повторяющиеся рёбра не добавляются;
- пропускные способности задаются случайно.

Количество дополнительных рёбер:

```cpp
extraEdges = n * 8;
```

Итоговое количество логических рёбер:

```text
(n - 1) + 8n = 9n - 1
```

Для каждого логического ребра в остаточной сети хранится прямое и обратное ребро, поэтому в памяти хранится:

```text
2 * (9n - 1)
```

объектов `Edge`.

## Benchmark

По умолчанию тестируются графы до 4000 вершин:

```cpp
vector<int> sizes = {
    50, 100, 150, 200, 250,
    300, 350, 400, 450, 500,
    600, 700, 800, 900, 1000,
    1200, 1400, 1600, 1800, 2000,
    2500, 3000, 3500, 4000
};
```

Для каждого размера выполняется несколько запусков, после чего считается среднее время работы.

По умолчанию:

```cpp
testsPerSize = 100;
```

Количество тестов можно передать первым аргументом командной строки.

## Как запустить

### Вариант 1. Через g++

Создать папку для результатов:

```bash
mkdir -p results
```

Скомпилировать:

```bash
g++ -O2 -std=c++17 src/main.cpp -o maxflow_benchmark
```

Запустить быстрый тест, например 5 запусков на размер:

```bash
./maxflow_benchmark 5
```

Запустить полный benchmark на 100 запусков:

```bash
./maxflow_benchmark 100
```

На Windows запуск может выглядеть так:

```powershell
mkdir results
g++ -O2 -std=c++17 src/main.cpp -o maxflow_benchmark.exe
.\maxflow_benchmark.exe 5
```

### Вариант 2. Через CMake

```bash
cmake -S . -B build
cmake --build build --config Release
```

Запуск:

```bash
./build/maxflow_benchmark 5
```

На Windows исполняемый файл может находиться в:

```text
build/Release/maxflow_benchmark.exe
```

## Вывод

Эксперименты показывают, что на сложных графах алгоритм Диница обычно работает быстрее алгоритмов Форда-Фалкерсона и Эдмондса-Карпа.

Это связано с тем, что Диниц использует слоистую сеть и блокирующий поток. В отличие от алгоритмов, которые увеличивают поток по одному пути, Диниц за одну фазу может использовать сразу несколько путей.
